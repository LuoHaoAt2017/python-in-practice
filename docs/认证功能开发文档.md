# JWT 登录认证功能开发文档

## 1. 功能概述

为 Sakila API 添加基于 JWT（JSON Web Token）的登录认证机制，保护所有业务接口（actors / films / customers / rentals），health 检查接口不需要认证。

---

## 2. 技术选型

| 组件 | 选用库 | 说明 |
|---|---|---|
| 密码哈希 | `passlib[bcrypt]` + `bcrypt==4.0.1` | bcrypt 算法，不可逆加密存储 |
| JWT 签发/校验 | `python-jose[cryptography]` | HS256 对称签名，FastAPI 官方推荐 |
| 邮箱校验 | `email-validator` | Pydantic `EmailStr` 字段依赖 |

## 4. 数据库模型

### User 表（`users`）

```python
class User(Base, TimestampMixin):
    __tablename__ = "users"

    user_id: int         # PK，自增
    username: str        # 唯一，最长 50 字符，索引
    email: str           # 唯一，最长 255 字符，索引
    hashed_password: str # bcrypt 哈希，最长 255 字符
    is_active: bool      # 是否启用，默认 True
    is_superuser: bool   # 是否超级用户，默认 False
    created_at: datetime # 自动填充
    updated_at: datetime # 自动更新
```

> User 模型为独立认证用户，不依赖 Sakila 的 Staff / Address / Store 表。

## 5. 接口说明

### 5.1 注册

```
POST /api/v1/auth/register
```

**请求体：**
```json
{
  "username": "testuser",
  "email": "test@example.com",
  "password": "password123"
}
```

- `password` 最短 8 位
- `username` 和 `email` 全局唯一

**成功响应（201）：**
```json
{
  "success": true,
  "code": 201,
  "message": "Created",
  "data": {
    "user_id": 1,
    "username": "testuser",
    "email": "test@example.com",
    "is_active": true,
    "created_at": "2026-04-06T03:35:46Z",
    "updated_at": "2026-04-06T03:35:46Z"
  }
}
```

**错误响应：**
- `409 Conflict` — 用户名或邮箱已被注册
- `422 Unprocessable Entity` — 字段校验失败（如密码不足 8 位）

---

### 5.2 登录

```
POST /api/v1/auth/login
```

**请求体：**
```json
{
  "username": "testuser",
  "password": "password123"
}
```

**成功响应（200）：**
```json
{
  "success": true,
  "code": 200,
  "message": "Success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "bearer"
  }
}
```

> 客户端需从 `data.access_token` 中取出 token。

**错误响应：**
- `401 Unauthorized` — 用户名或密码错误

---

### 5.3 访问受保护接口

在请求头中携带 Bearer token：

```
Authorization: Bearer <access_token>
```

**示例：**
```bash
curl http://localhost:8000/api/v1/actors/ \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

**401 场景：**
- 未携带 token
- token 格式错误或签名无效
- token 已过期
- token 对应用户不存在或已禁用

## 6. 路由保护策略

在 `api/router.py` 的 `include_router` 层面统一注入依赖，无需修改各 endpoint 函数：

```python
# 公开路由（无认证）
api_router.include_router(auth.router, prefix="/auth", tags=["auth"])
api_router.include_router(health.router, tags=["health"])

# 受保护路由（需 JWT）
api_router.include_router(
    actors.router, prefix="/actors", tags=["actors"],
    dependencies=[Depends(get_current_user)],
)
# films / customers / rentals 同上
```

## 7. 环境变量配置

在 `.env` 中可覆盖以下配置：

```env
SECRET_KEY=your-production-secret-key-here
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

> **生产环境必须替换 `SECRET_KEY`**，使用足够随机的长字符串（至少 32 位）。

## 8. 完整认证流程

```
客户端                              服务端
  │                                   │
  │── POST /auth/register ──────────► │  创建用户，密码 bcrypt 哈希存储
  │◄─ 201 UserResponse ───────────── │
  │                                   │
  │── POST /auth/login ────────────► │  查询用户，验证密码哈希
  │◄─ 200 { access_token } ──────── │  签发 JWT（sub=username, exp=30min）
  │                                   │
  │── GET /actors/                    │
  │   Authorization: Bearer <token> ► │  decode_access_token → get_user_by_username
  │◄─ 200 actors list ─────────────  │  验证通过，返回数据
```

---

## 9. 安全说明

- 密码使用 **bcrypt** 单向哈希，数据库中不存储明文
- JWT 使用 **HS256** 对称签名，密钥来自 `SECRET_KEY` 环境变量
- Token 默认有效期 **30 分钟**，过期后需重新登录
- 注册时同时校验 `username` 和 `email` 唯一性，防止枚举攻击的信息泄露可通过返回统一 "用户名或邮箱已存在" 进一步加固

---

## 10. 快速验证

```bash
# 注册
curl -s -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","email":"admin@example.com","password":"admin1234"}'

# 登录，取 token
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin1234"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['access_token'])")

# 访问受保护接口
curl http://localhost:8000/api/v1/films/ \
  -H "Authorization: Bearer $TOKEN"

# 无 token 应返回 401
curl http://localhost:8000/api/v1/films/

# health 无需 token
curl http://localhost:8000/api/v1/health
```
