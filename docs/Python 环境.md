什么是 Python 环境？为什么需要环境？怎么用？

你现在所有的疑惑，都会一次性解决。

---

# 一、什么是 Python 环境？
一句话：
**Python 环境 = 一个装了 Python + 各种库（依赖）的独立“盒子”**

你可以把它理解成：
**一个独立的工具箱。**

里面包含：
- Python 解释器（能运行代码）
- 你安装的库（fastapi、flask 等）
- 库的版本
- 配置

---

# 二、Python 有几种环境？（最重要）
Python 环境一共就 **3 种**，你必须分清：

## 1. 系统环境（全局环境）
你安装 Python 时，系统自动创建的。
**所有项目共用这一个环境。**

问题：
项目A要 flask 1.0
项目B要 flask 2.0
→ **会冲突！**

所以：**不要用系统环境开发项目！**

---

## 2. 虚拟环境（推荐！项目专用）
**给每个项目单独创建一个独立环境。**

特点：
- 环境之间完全隔离
- 装的库互不影响
- 干净、不会乱
- 企业开发标准

你之前学的：
```bash
python -m venv venv
```
就是创建**虚拟环境**。

---

## 3. conda 环境（数据科学/AI 用）
适合数据分析、机器学习，后端一般不用。

---

# 三、为什么一定要用虚拟环境？（核心）
我给你举个最真实的例子：

### 项目1
用：flask 1.0

### 项目2
用：flask 2.0

如果不隔离：
- 安装 flask 2.0 会把 1.0 覆盖
- 项目1 直接报错崩溃

**虚拟环境 = 每个项目有自己独立房间，互不干扰**

---

# 四、虚拟环境工作原理（超简单）
1. 你创建一个环境：`venv`
2. 激活它：`(venv)` 出现
3. 你 `pip install` 安装的库
   → **全部放进这个 venv 文件夹里**
4. 退出环境后，就无法使用这些库
5. 每个项目都有自己的 venv

→ **这就是为什么你在项目里看不到依赖！**
依赖都在 `venv` 文件夹里，不是在代码文件夹里。

---

# 五、最常用虚拟环境命令（必背）
## 1. 创建环境
```bash
python -m venv venv
```

## 2. 激活环境
Windows：
```bash
venv\Scripts\activate
```

Mac/Linux：
```bash
source venv/bin/activate
```

激活成功标志：
```
(venv) X:\项目路径>
```

## 3. 在这个环境里安装库
```bash
pip install fastapi
```
只安装在当前虚拟环境，不影响其他项目。

## 4. 退出环境
```bash
deactivate
```

## 5. 删除环境
直接删除 `venv` 文件夹即可。

# 六、你最关心的问题：
## Q：为什么我安装了依赖，项目里看不到？
A：
**依赖安装在虚拟环境里（venv 文件夹），
不是在你的代码文件夹里。**

代码只需要：
```python
import fastapi
```
Python 会自动去虚拟环境里找。

---

# 七、最简单总结（你只需要记住这 4 条）
1. **系统环境**：公共环境，不要用来开发
2. **虚拟环境**：每个项目一个，独立隔离（必须用）
3. **激活环境**才能安装、使用依赖
4. **依赖存在环境里，项目里看不到是正常的**

---

# 八、一句话终极总结
**开发 Python 项目 = 必须先创建虚拟环境，再激活，再安装依赖！**
