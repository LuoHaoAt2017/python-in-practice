# 系统技术选型分析

## 项目概述

**Python版本**: 3.11+
**包管理工具**: uv (现代Python包管理器)

---

## 核心技术栈

### 1. Web框架与服务器

#### FastAPI 0.109.2
- **用途**: 主Web框架，提供RESTful API
- **特点**:
  - 基于Starlette的高性能异步框架
  - 自动生成OpenAPI文档
  - 内置数据验证（基于Pydantic）
  - 原生支持异步编程
- **应用场景**: 所有HTTP API接口

#### Uvicorn 0.27.1
- **用途**: ASGI服务器
- **特点**: 轻量级、高性能的ASGI实现
- **部署模式**: 作为Gunicorn的Worker进程

#### Gunicorn 20.1.0
- **用途**: 生产环境应用服务器
- **配置**:
  - Worker数量: 4个（可通过环境变量SERVER_WORKERS配置）
  - Worker类型: UvicornWorker
  - 超时时间: 300秒
- **特点**:
  - 多进程架构，提高并发处理能力
  - 支持优雅重启
  - 统一JSON日志格式输出

---

### 2. 数据库层

#### PostgreSQL
- **用途**: 主数据库
- **特点**:
  - 企业级关系型数据库
  - 支持复杂查询和事务

#### SQLAlchemy 1.4.54
- **用途**: ORM框架
- **特点**:
  - 支持异步操作（AsyncSession）
  - 连接池管理（pool_size=100, max_overflow=100）
  - 连接回收（pool_recycle=1800秒）
  - 连接健康检查（pool_pre_ping=True）

#### asyncpg 0.30.0
- **用途**: PostgreSQL异步驱动
- **优势**: 高性能异步数据库操作

#### psycopg2-binary 2.9.10
- **用途**: PostgreSQL同步驱动
- **场景**: 数据库初始化、迁移等同步操作

#### Alembic 1.4.3
- **用途**: 数据库迁移工具
- **特点**:
  - 版本控制数据库schema
  - 支持向上/向下迁移
  - 集成到应用启动流程

---

### 3. 异步与任务处理

#### Celery 5.4.0
- **用途**: 分布式任务队列
- **特点**:
  - 异步任务处理
  - 定时任务调度
  - 任务结果存储
- **应用场景**:
  - 数据导出
  - 批量数据处理
  - 定时统计任务

#### Redis 3.5.3
- **用途**:
  - Celery消息队列broker
  - 缓存层
  - 任务结果存储
- **集成**: 通过OpenTelemetry自动埋点监控

#### asyncio
- **用途**: Python原生异步编程
- **应用**:
  - 异步数据库操作
  - 并发请求处理
  - 后台调度器

#### Gevent 22.10.2
- **用途**: 协程库
- **特点**: 基于libev的协程实现

---

### 4. 数据处理与验证

#### Pydantic 2.11.4
- **用途**: 数据验证和序列化
- **特点**:
  - 类型提示驱动的数据验证
  - 自动序列化/反序列化
  - 性能优化（基于Rust实现）
- **应用**:
  - API请求/响应模型
  - 配置文件解析
  - 数据库实体映射

#### PyYAML 6.0.2
- **用途**: YAML配置文件解析
- **应用**:
  - 应用配置（app_config.yaml）
  - 质控规则配置
  - 过滤器配置

#### openpyxl 3.0.10
- **用途**: Excel文件处理
- **场景**: 质控规则导入、数据导出

#### beautifulsoup4
- **用途**: HTML/XML解析
- **场景**: 病历文书内容解析（HTML格式）

---

### 5. HTTP客户端与RPC

#### httpx 0.28.1
- **用途**: 现代HTTP客户端
- **特点**:
  - 支持异步操作
  - HTTP/2支持
  - 连接池管理
- **应用**: 调用第三方API服务

#### grpcio 1.68.1
- **用途**: gRPC框架
- **场景**: 微服务间RPC通信
- **特点**: 高性能、强类型的RPC调用

---

### 6. 可观测性（Observability）

#### OpenTelemetry
- **用途**: 链路追踪、监控
- **集成**:
  - FastAPI自动埋点
  - Redis调用监控
  - Celery任务监控
- **配置**: otel_config.yaml
- **日志**: 统一JSON格式输出

#### 日志系统
- **实现**: 自定义JsonFormatter
- **特点**:
  - 结构化日志输出
  - 统一日志格式（gunicorn、应用日志）
  - 包含runtime_app、runtime_module等字段

---

### 7. 认证与安全

#### JWT认证
- **实现**: 自定义JWT验证器
- **特点**:
  - 异步初始化公钥
  - 应用启动时加载
- **异常处理**: 统一返回401状态码

#### CORS
- **配置**: 允许所有源（allow_origins=["*"]）
- **方法**: 允许所有HTTP方法
- **凭证**: 支持携带凭证

---

### 8. 部署与运维

#### Docker
- **基础镜像**: python:3.11.10-slim-bookworm-build
- **时区**: Asia/Shanghai
- **部署路径**: /root/medical_record_qc

#### CI/CD
- **工具**: Jenkins Pipeline
- **流程**:
  1. 代码拉取（GitLab）
  2. 配置文件合并
  3. Docker镜像构建
  4. Ansible自动化部署

#### 多进程架构
- **主进程**: Gunicorn Master进程
- **Worker进程**: 4个UvicornWorker处理HTTP请求
- **Scheduler进程**: 独立进程运行定时任务
- **DataProcessor进程**: 独立进程处理数据流

---

## 技术架构特点

### 1. 异步优先
- 全面采用async/await语法
- 异步数据库操作
- 异步HTTP客户端
- 提高并发处理能力

### 2. 多进程架构
```
┌─────────────────────────────────────┐
│         Gunicorn Master            │
└──────────────┬──────────────────────┘
               │
       ┌───────┴────────┬─────────────┐
       │                │             │
  ┌────▼────┐      ┌───▼────┐   ┌───▼─────┐
  │ Worker1 │      │Worker2 │   │ Worker3 │ ...
  │Uvicorn  │      │Uvicorn │   │Uvicorn  │
  └─────────┘      └────────┘   └─────────┘
       │
       ├──────────────────────────────┐
       │                              │
  ┌────▼─────────┐            ┌──────▼──────┐
  │  Scheduler   │            │DataProcessor│
  │   Process    │            │   Process   │
  └──────────────┘            └─────────────┘
```

### 3. 领域驱动设计
- **api层**: 路由和请求处理
- **core层**: 业务逻辑核心
  - datasource: 数据源管理
  - tasks: 任务调度
  - data_processor: 数据处理
  - statistics: 统计分析
- **db层**: 数据访问层
- **utils层**: 工具类

### 4. 配置管理
- **配置文件**: YAML格式
- **分层配置**:
  - 应用配置（app_config.yaml）
  - 质控规则配置（qc_configs/）
  - 过滤器配置（filter_*.json/yaml）
- **环境变量**: 支持运行时配置覆盖

## 总结

该项目采用了现代化的Python技术栈，以FastAPI为核心框架，全面拥抱异步编程，结合Celery分布式任务队列、PostgreSQL关系型数据库、Redis缓存等企业级组件，构建了一个高性能、可扩展、易维护的病历质控系统后端服务。

**核心技术亮点**:
1. 异步优先的架构设计
2. 多进程并发处理
3. 完善的可观测性（OpenTelemetry）
4. 企业级部署方案（Docker + Jenkins + Ansible）
5. 类型安全的开发体验（Pydantic + Type Hints）

该技术选型适合中大型企业级应用，能够支撑高并发、大数据量的业务场景。
