# Python 特殊变量 `__name__` 详解
`__name__` 是 Python 内置的**特殊双下划线变量**（也叫魔法变量），核心作用是**标识当前模块（.py 文件）的运行方式**：是**直接运行**这个文件，还是**被其他文件导入**使用。

它是 Python 区分「脚本执行」和「模块导入」的关键，也是写可复用、可测试代码的核心知识点。

---

## 一、核心规则（记住这1条就够了）
Python 会自动给 `__name__` 赋值，规则只有两个：
1. **直接运行当前 .py 文件** → `__name__ = "__main__"`
2. **当前文件被其他文件导入** → `__name__ = "当前文件的文件名（不带.py后缀）"`

---

## 二、直观代码示例
我们创建两个文件，一看就懂：

### 1. 测试文件 1：`demo.py`
```python
# demo.py
# 直接打印 __name__ 的值
print(f"demo.py 中的 __name__ 值是：{__name__}")
```

#### 情况1：直接运行 `demo.py`
在终端执行：
```bash
python demo.py
```
输出结果：
```
demo.py 中的 __name__ 值是：__main__
```

#### 情况2：被其他文件导入
新建文件 `test.py`：
```python
# test.py
# 导入 demo 模块
import demo
```
运行 `test.py`：
```bash
python test.py
```
输出结果：
```
demo.py 中的 __name__ 值是：demo
```

---

## 三、最经典用法：`if __name__ == "__main__"`
这是 `__name__` 99% 的使用场景，作用是：
**只有直接运行当前文件时，才执行这段代码；被导入时不执行。**

### 示例：可复用的工具函数
`utils.py`（既可以当工具模块导入，也能直接运行测试）：
```python
# utils.py
def add(a, b):
    """加法工具函数"""
    return a + b

# 核心：直接运行时才执行测试代码
if __name__ == "__main__":
    # 测试函数功能
    print("测试加法函数：")
    print(f"1 + 2 = {add(1, 2)}")
```

### 两种运行效果
1. **直接运行 `utils.py`** → 执行测试代码，输出结果
2. **被其他文件导入** → 只提供 `add` 函数，**不执行测试代码**

```python
# 其他文件导入使用
import utils
print(utils.add(5, 6))  # 只会输出 11，不会执行测试代码
```

---

## 四、为什么要用这个语法？（3个核心好处）
1. **代码复用**：同一个 `.py` 文件，既能当独立脚本运行，又能当模块被其他文件导入
2. **避免冲突**：被导入时，不会自动执行测试代码、主逻辑代码，防止意外运行
3. **规范代码结构**：让程序的「主入口」更清晰，符合 Python 编码规范

---

## 五、补充知识点
1. **主程序文件**：Python 程序的入口文件，永远满足 `__name__ == "__main__"`
2. **内置模块**：导入 Python 自带模块（如 `os`、`sys`），`__name__` 就是模块名
3. **包导入**：如果是包里面的模块，`__name__` 会包含包名（如 `package.demo`）

---

### 总结
1. `__name__` 是 Python 自动赋值的特殊变量，**无需手动定义**
2. 直接运行文件 → `__name__ == "__main__"`；被导入 → `__name__ == 文件名`
3. 核心用法：`if __name__ == "__main__":` 包裹主逻辑/测试代码，实现文件两用
