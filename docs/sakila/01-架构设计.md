# Sakila数据库项目 - 架构设计

## 概述

Sakila数据库项目是一个基于FastAPI和PostgreSQL构建的现代化DVD租赁系统API。本项目采用分层架构设计，遵循领域驱动设计原则，提供完整的RESTful API接口。

## 技术栈

### 核心框架
- **FastAPI**: 现代、快速的Python Web框架，支持异步/等待，自动生成OpenAPI文档
- **SQLAlchemy**: Python SQL工具包和对象关系映射(ORM)系统
- **Pydantic**: 数据验证和序列化库

### 数据库
- **PostgreSQL**: 关系型数据库管理系统
- **Asyncpg**: PostgreSQL异步数据库驱动

### 开发工具
- **Python 3.11+**: 编程语言
- **Uvicorn**: ASGI服务器
- **Pytest**: 测试框架

## 架构设计原则

### 1. 分层架构
项目采用经典的三层架构模式：

```
表示层 (Presentation Layer)
    ├── API端点 (api/endpoints/)
    └── 请求/响应模型 (schemas/)

业务逻辑层 (Business Logic Layer)
    └── 服务层 (services/)

数据访问层 (Data Access Layer)
    ├── ORM模型 (models/)
    └── 数据库配置 (config/database.py)
```

### 2. 依赖注入
- 使用FastAPI的`Depends`实现依赖注入
- 数据库会话通过依赖注入传递到各层
- 服务层可独立测试和重用

### 3. 异步优先
- 所有数据库操作都使用异步模式
- API端点支持async/await语法
- 使用SQLAlchemy的异步会话

## 各层职责说明

### 1. API端点层 (`api/`)
- 处理HTTP请求和响应
- 定义路由和端点
- 输入参数验证
- 调用服务层处理业务逻辑
- 错误处理和状态码返回

### 2. 服务层 (`services/`)
- 封装核心业务逻辑
- 协调多个模型的操作
- 事务管理
- 数据验证和转换
- 可重用业务组件

### 3. 数据模型层 (`models/`)
- 定义数据库表结构
- ORM映射配置
- 数据关系定义
- 数据约束和验证
- 查询方法封装

### 4. 数据验证层 (`schemas/`)
- 请求/响应数据结构定义
- 数据验证规则
- 序列化/反序列化
- 文档生成支持
- API契约定义

### 5. 配置层 (`config/`)
- 应用配置管理
- 数据库连接配置
- 环境变量管理
- 安全配置

## 设计模式应用

### 1. 仓储模式 (Repository Pattern)
- 服务层提供类似仓储的接口
- 封装数据访问逻辑
- 提供高级查询方法

### 2. 依赖倒置原则 (DIP)
- 高层模块不依赖低层模块
- 抽象不依赖具体实现
- 通过接口进行解耦

### 3. 单一职责原则 (SRP)
- 每个类/函数只负责一个职责
- 端点只处理HTTP相关逻辑
- 服务只处理业务逻辑
- 模型只处理数据表示

### 4. 开放封闭原则 (OCP)
- 对扩展开放，对修改封闭
- 通过继承和组合扩展功能
- 减少对现有代码的修改

## 异步架构设计

### 1. 异步数据库操作
```python
# 使用异步会话管理
async def get_actors(db: AsyncSession):
    result = await db.execute(select(Actor))
    return result.scalars().all()
```

### 2. 异步端点
```python
# FastAPI支持异步端点
@router.get("/{actor_id}")
async def get_actor(actor_id: int, db: AsyncSession = Depends(get_async_db)):
    actor = await ActorService.get_actor_by_id(db, actor_id)
    return actor
```

### 3. 异步生命周期管理
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动时初始化
    async with async_engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    yield
    # 关闭时清理
    await async_engine.dispose()
```
