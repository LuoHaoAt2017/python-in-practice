# Sakila数据库项目 - 服务层设计

## 概述

服务层（Service Layer）是Sakila项目的业务逻辑核心，负责协调数据访问、业务规则实施和事务管理。服务层位于API端点和数据模型之间，实现了关注点分离，使得业务逻辑可重用、可测试。

## 服务层架构设计

### 1. 服务层定位

```
API端点层 (Presentation Layer)
    ↓
服务层 (Business Logic Layer)  ← 本层
    ↓
数据访问层 (Data Access Layer)
```

### 2. 设计原则

- **单一职责原则**: 每个服务类专注于一个业务实体
- **依赖倒置原则**: 服务层不直接依赖具体实现
- **开闭原则**: 易于扩展，无需修改现有代码
- **事务完整性**: 确保业务操作的事务性

## 服务类设计模式

### 1. 静态方法模式

所有服务类都采用静态方法设计，避免状态管理复杂性：

```python
class ActorService:
    """Service for actor-related operations."""

    @staticmethod
    async def get_actors(
        db: AsyncSession,
        skip: int = 0,
        limit: int = 100,
        name: Optional[str] = None
    ) -> List[Actor]:
        """Get list of actors with optional filtering and pagination."""
        # 实现逻辑
```

### 2. 统一接口模式

每个服务类都实现类似的CRUD操作接口：

- `get_<entity>s()` - 获取列表（支持分页和过滤）
- `get_<entity>_by_id()` - 根据ID获取单个实体
- `create_<entity>()` - 创建新实体
- `update_<entity>()` - 更新现有实体
- `delete_<entity>()` - 删除实体
- `get_<entity>s_count()` - 获取总数（用于分页）

### 3. 依赖注入模式

服务方法通过参数接收数据库会话：

```python
@staticmethod
async def get_actors(db: AsyncSession, ...):
    # 使用注入的db会话
    result = await db.execute(query)
    return result.scalars().all()
```

## 服务方法详细分析

### 1. 查询服务方法

#### 基本查询模式
```python
@staticmethod
async def get_actors(
    db: AsyncSession,
    skip: int = 0,
    limit: int = 100,
    name: Optional[str] = None
) -> List[Actor]:
    """获取演员列表，支持分页和过滤"""
    query = select(Actor).order_by(Actor.last_name, Actor.first_name)

    # 过滤条件
    if name:
        query = query.where(
            (Actor.first_name.ilike(f"%{name}%")) |
            (Actor.last_name.ilike(f"%{name}%"))
        )

    # 分页
    if limit > 0:
        query = query.offset(skip).limit(limit)

    result = await db.execute(query)
    return result.scalars().all()
```

#### 复杂过滤查询（FilmService示例）
```python
@staticmethod
async def get_films(
    db: AsyncSession,
    skip: int = 0,
    limit: int = 100,
    title: Optional[str] = None,
    release_year: Optional[int] = None,
    rating: Optional[str] = None,
    language_id: Optional[int] = None,
) -> List[Film]:
    """获取电影列表，支持多种过滤条件"""
    query = select(Film).order_by(Film.title)

    filters = []
    if title:
        filters.append(Film.title.ilike(f"%{title}%"))
    if release_year:
        filters.append(Film.release_year == release_year)
    if rating:
        filters.append(Film.rating == rating)
    if language_id:
        filters.append(Film.language_id == language_id)

    if filters:
        query = query.where(and_(*filters))

    if limit > 0:
        query = query.offset(skip).limit(limit)

    result = await db.execute(query)
    return result.scalars().all()
```

#### 关联数据查询

### 3. 更新服务方法

#### 通用更新模式
```python
@staticmethod
async def update_actor(
    db: AsyncSession,
    actor_id: int,
    actor_data: ActorUpdate
) -> Optional[Actor]:
    """更新现有演员"""
    # 1. 获取现有实体
    actor = await ActorService.get_actor_by_id(db, actor_id)
    if not actor:
        return None

    # 2. 应用更新数据
    update_data = actor_data.model_dump(exclude_unset=True)
    for field, value in update_data.items():
        setattr(actor, field, value)

    # 3. 提交事务
    await db.commit()
    await db.refresh(actor)
    return actor
```

#### 关键特性
- `exclude_unset=True`: 只更新客户端提供的字段
- 属性动态设置：支持灵活的字段更新
- 自动刷新：确保返回最新数据

### 4. 删除服务方法

## 查询优化技术

### 1. 预加载（Eager Loading）

#### 关联数据预加载
```python
@staticmethod
async def get_customer_with_rentals(db: AsyncSession, customer_id: int) -> Optional[Customer]:
    """获取客户及其租赁记录"""
    query = (
        select(Customer)
        .where(Customer.customer_id == customer_id)
        .options(
            selectinload(Customer.rentals).selectinload("inventory").selectinload("film"),
            selectinload(Customer.payments),
        )
    )
    result = await db.execute(query)
    return result.scalar_one_or_none()
```

#### 预加载策略
- `selectinload()`: 适合一对多关系
- `joinedload()`: 适合一对一或少量关联
- 避免N+1查询问题

### 2. 复杂查询构建

#### 动态过滤构建
```python
@staticmethod
async def get_rentals(
    db: AsyncSession,
    skip: int = 0,
    limit: int = 100,
    customer_id: Optional[int] = None,
    inventory_id: Optional[int] = None,
    staff_id: Optional[int] = None,
    returned: Optional[bool] = None,
    start_date: Optional[datetime] = None,
    end_date: Optional[datetime] = None,
) -> List[Rental]:
    """获取租赁列表，支持多种过滤条件"""
    query = select(Rental).order_by(desc(Rental.rental_date))

    filters = []
    if customer_id:
        filters.append(Rental.customer_id == customer_id)
    if inventory_id:
        filters.append(Rental.inventory_id == inventory_id)
    if staff_id:
        filters.append(Rental.staff_id == staff_id)

    # 业务逻辑：归还状态过滤
    if returned is not None:
        if returned:
            filters.append(Rental.return_date.is_not(None))  # 已归还
        else:
            filters.append(Rental.return_date.is_(None))    # 未归还

    # 时间范围过滤
    if start_date:
        filters.append(Rental.rental_date >= start_date)
    if end_date:
        filters.append(Rental.rental_date <= end_date)

    if filters:
        query = query.where(and_(*filters))

    if limit > 0:
        query = query.offset(skip).limit(limit)

    result = await db.execute(query)
    return result.scalars().all()
```

## 错误处理模式

### 1. 返回空值模式
```python
@staticmethod
async def get_actor_by_id(db: AsyncSession, actor_id: int) -> Optional[Actor]:
    """根据ID获取演员，不存在时返回None"""
    query = select(Actor).where(Actor.actor_id == actor_id)
    result = await db.execute(query)
    return result.scalar_one_or_none()  # 返回None而不是抛出异常
```

### 2. 布尔结果模式
```python
@staticmethod
async def delete_actor(db: AsyncSession, actor_id: int) -> bool:
    """删除演员，返回是否成功"""
    actor = await ActorService.get_actor_by_id(db, actor_id)
    if not actor:
        return False  # 不存在，删除失败

    await db.delete(actor)
    await db.commit()
    return True  # 删除成功
```

### 3. 异常处理策略
- 服务层不直接处理HTTP异常
- 返回适当的指示符（None, False等）
- API端点层负责转换业务结果为HTTP响应

## 事务管理

### 1. 显式事务控制
```python
@staticmethod
async def create_actor(db: AsyncSession, actor_data: ActorCreate) -> Actor:
    """创建新演员"""
    actor = Actor(...)
    db.add(actor)

    # 显式提交事务
    await db.commit()

    # 刷新以获取数据库生成的值（如ID）
    await db.refresh(actor)
    return actor
```

### 2. 事务完整性
- 每个操作都是原子性的
- 失败时自动回滚
- 确保数据一致性

### 2. 服务组合模式
```python
class ComplexBusinessService:
    """组合多个简单服务完成复杂业务逻辑"""

    @staticmethod
    async def rent_film(db: AsyncSession, customer_id: int, film_id: int):
        # 1. 检查客户状态
        customer = await CustomerService.get_customer_by_id(db, customer_id)
        if not customer.activebool:
            raise BusinessError("Customer is inactive")

        # 2. 查找可用库存
        inventory = await InventoryService.find_available_inventory(db, film_id)
        if not inventory:
            raise BusinessError("Film not available")

        # 3. 创建租赁记录
        rental = await RentalService.create_rental(
            db,
            inventory_id=inventory.inventory_id,
            customer_id=customer_id,
            staff_id=1  # 默认员工
        )

        # 4. 更新库存状态
        await InventoryService.mark_as_rented(db, inventory.inventory_id)

        return rental
```

## 总结

Sakila项目的服务层设计体现了现代企业级应用的最佳实践：

1. **清晰的职责分离**: 服务层专注于业务逻辑，与API和数据访问解耦
2. **一致的设计模式**: 所有服务遵循相同的接口和实现模式
3. **事务完整性**: 确保业务操作的原子性和一致性
4. **异步优先**: 支持高并发，性能优异
5. **测试友好**: 纯函数特性，易于单元测试
6. **扩展性强**: 易于添加新服务和组合现有服务

这种服务层设计不仅提供了强大的业务逻辑处理能力，还为项目的长期维护和扩展奠定了坚实基础。通过服务层的抽象，业务规则可以独立于API和数据存储实现，大大提高了系统的灵活性和可维护性。
