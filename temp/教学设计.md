# 超详细版：什么是教学设计？考什么？怎么写？怎么拿高分？
（专为**高中信息技术科目三**+**非师范、开发背景**讲解，全程通俗、不搞教育学黑话）

---

# 一、先把定义说透：教学设计到底是啥？
**教学设计 = 你对“一堂信息技术课”的完整上课方案**
官方一点说：
为了达成教学目标，对**教什么、怎么教、怎么练、怎么评**进行系统规划的过程。

放到科目三里，简单理解：
**最后一道大题 = 写一篇标准教案
分值大约 25～35 分
是科目三**分值最高、决定过不过**的一道题**

它不考你技术有多深，
考的是：**你会不会像老师一样上课**。

---

# 二、科目三“教学设计题”长什么样？
试卷会给你一段材料，通常是：
- 某节课的教材内容片段（比如 Python 循环、信息安全、数据编码、IP 地址等）
- 要求：**根据材料完成一篇完整的教学设计**

你要写的内容是**固定 7 大部分**，一个都不能少。

---

# 三、一篇完整教学设计必须包含的 7 大模块（必考结构）
## 1. 基本信息（送分，30 秒写完）
- 课题：××××（题目抄过来）
- 课时：1 课时
- 课型：新授课 / 实操课
- 授课对象：高中一年级学生

## 2. 教学目标（必须按“四大核心素养”写）
这是信息技术独有的评分点，**不写就扣分**。
四个必须全写：

1）**信息意识**
能感知信息、数据的存在；能判断信息价值、真伪、安全性。

2）**计算思维**
能对问题抽象、分解、建模；能用算法/程序实现自动化解决。

3）**数字化学习与创新**
能使用信息技术工具完成任务；能优化方法、尝试创新。

4）**信息社会责任**
遵守网络道德与法规；保护隐私；安全、负责任地使用技术。

你考试时**直接套句式**，不用自己编。

## 3. 教学重难点
- **重点**：本节课必须掌握的知识点/操作（比如 for 循环语法、IP 格式）
- **难点**：学生容易懵、容易错的地方（如循环条件、变量变化、逻辑理解）

## 4. 教学方法
写 3～4 个就行，信息技术高频：
- 任务驱动法（最常用、必写）
- 小组合作学习
- 探究式学习
- 演示法
- 实操练习法

## 5. 教学过程（最重要！占 70% 分数）
必须按固定流程写，**一步都不能跳**：

### （1）情境导入（3～5 分钟）
用生活例子/问题/场景引出本节课内容。
目的：吸引学生、点明本节课要解决什么问题。

### （2）新知探究/讲授（10 分钟左右）
教师讲解核心概念 + 演示关键操作。
比如讲 Python，就演示一段代码运行过程。

### （3）任务实践（15～20 分钟）
**信息技术课的灵魂：必须有实操！**
最好写**分层任务**（体现你会教学）：
- 任务 1：基础任务（所有人都会）
- 任务 2：提高任务（巩固）
- 任务 3：拓展任务（学有余力）

### （4）成果展示与评价（5 分钟）
学生展示作品 → 自评 → 互评 → 教师点评
体现“过程性评价”。

### （5）课堂小结（2 分钟）
回顾本节课知识点，再次点核心素养。

### （6）布置作业
分两层：
- 基础作业：巩固操作
- 拓展作业：生活应用/小项目

## 6. 板书设计
简单列关键词即可，不用美术字。
比如：
1. 概念
2. 语法/步骤
3. 注意事项

## 7. 教学反思（简单两句）
写：
- 可能存在学生操作不熟练
- 需加强分层指导
- 今后更突出计算思维

就够了。

# 四、评分老师到底怎么打分？（关键内幕）
阅卷老师不会逐字读，只扫**关键得分点**：

1. **结构完整吗？**（7 大项缺一项就扣分）
2. **目标写核心素养了吗？**（没写直接低分）
3. **有没有任务、有没有实操？**（信息技术课必须动手）
4. **有没有学生活动、评价环节？**
5. **语言是否像老师，而不是程序员写教程？**

你是开发出身，最容易犯的错：
**写得太技术、太硬核、没有学生活动、全是知识点堆砌**
这样直接丢 15+ 分。

# 五、用一个例子让你彻底看懂
以课题：**《Python for 循环语句》**

### 教学目标（套写示范）
1. 信息意识：能识别重复执行类问题，理解循环的应用场景。
2. 计算思维：能将重复操作抽象为循环结构，实现自动化处理。
3. 数字化学习与创新：能使用 for 循环解决统计、累加等实际问题。
4. 信息社会责任：规范编写代码，养成良好编程习惯。

### 教学过程片段示范
- 导入：统计 1～100 累加，手动算很慢 → 引出循环
- 新知：讲解 for 循环语法 + 演示简单案例
- 任务：
  任务1：输出 1～10
  任务2：求 1～100 和
  任务3：统计成绩列表中的及格人数
- 展示：学生上台演示代码
- 小结：循环结构、计算思维
- 作业：完善程序 + 尝试改写 while 实现

你看，这就是**标准教学设计**，不是技术文档。

# 六、对你最重要的 3 条忠告
1. **教学设计 ≠ 写技术教程**
   不要堆代码、不讲原理、不讲学生怎么学。

2. **必须体现“学生活动+实操”**
   信息技术课=动手课，没有实操=低分。

3. **结构永远不变**
   你只需要背模板，考试填空即可。
